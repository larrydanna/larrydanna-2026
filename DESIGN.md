## Aesthetic Direction

**Warm Dark / Studio Amber**

Not the cold blue-black of every other developer portfolio. This palette pulls from C64 amber monitors, the warm wood of a bluegrass stage, and decades of craft. It reads "person with deep interests and good taste" — technical without being sterile, personal without being precious.

## Color Palette

```css
:root {
  --bg:         #0F0D0A;  /* Near-black with warm undertone — main background */
  --surface:    #1B1915;  /* Elevated cards, nav, sidebars */
  --surface-2:  #252118;  /* Nested surfaces, hover states */
  --text:       #F2EDE2;  /* Warm white — primary body text */
  --muted:      #9E9485;  /* Metadata, timestamps, secondary labels */
  --accent:     #D4863A;  /* Burnt amber — primary CTAs, links, highlights */
  --accent-alt: #C84B31;  /* Rust red — danger states, alternate accent */
  --success:    #6DB884;  /* Sage green — success states, tags */
  --border:     #2E2A24;  /* Subtle dividers, card borders */
}
```

### Usage Rules

- `--accent` (#D4863A): primary buttons, active nav links, hover underlines, left borders on tip items, tag backgrounds
- `--accent-alt` (#C84B31): danger/destructive actions only; secondary accent when amber needs company
- `--success` (#6DB884): success messages, "vanilla ✓" badges in toy audit, positive status indicators
- `--muted` (#9E9485): dates, word counts, tag counts, placeholder text — never for anything interactive
- `--border` (#2E2A24): card outlines, horizontal rules, input borders — should barely register

## Motion

**Still.** This site does not animate.

- Hover state changes: instant (0ms transition — or 80ms max if needed for opacity)
- Hero section: one optional `fade-in` at 1.2s on first load only
- Nothing else moves, slides, bounces, or fades
- No scroll-triggered animations
- No loading spinners (pages load fast — design for that)

**Why:** Motion is a trust signal. Sites that animate everything feel like they're covering for weak content. This site's content is strong. Let it stand.

## Texture

Subtle CSS grain/noise on `--bg`. Applied via a pseudo-element or SVG filter — not a real image file.

```css
body::before {
  content: '';
  position: fixed;
  inset: 0;
  background-image: url("data:image/svg+xml,%3Csvg viewBox='0 0 256 256' xmlns='http://www.w3.org/2000/svg'%3E%3Cfilter id='noise'%3E%3CfeTurbulence type='fractalNoise' baseFrequency='0.9' numOctaves='4' stitchTiles='stitch'/%3E%3C/filter%3E%3Crect width='100%25' height='100%25' filter='url(%23noise)' opacity='0.04'/%3E%3C/svg%3E");
  opacity: 0.04;
  pointer-events: none;
  z-index: 0;
}
```

Opacity stays at or below 0.05. If it's visible at arm's length, it's too loud.

## Component Patterns

### Buttons

```css
/* Primary */
background: var(--accent);
color: #0F0D0A;
font: 500 15px/1 var(--font-body);
padding: 10px 20px;
border-radius: 4px;
border: none;

/* Secondary */
background: transparent;
color: var(--accent);
border: 1px solid var(--accent);

/* Ghost */
background: var(--surface);
color: var(--text);
border: 1px solid var(--border);
```

Border radius: 4px consistently. No pill buttons (too casual), no square (too rigid).

### Cards (Tool cards, blog cards)

```css
background: var(--surface);
border: 1px solid var(--border);
border-radius: 8px;
padding: 24px;
```

Hover: `border-color` shifts to `var(--accent)` at 40% opacity. No shadow, no lift — just the border brightens.

### Tip items

Left border accent pattern:

```css
border-left: 3px solid var(--accent);
padding-left: 16px;
```

### Tags / Badges

```css
background: rgba(212, 134, 58, 0.15);  /* --accent at 15% */
color: var(--accent);
font: 500 12px/1 var(--font-body);
padding: 4px 8px;
border-radius: 4px;
```

Success variant uses `--success` at 15% background.

### Navigation

- Fixed or sticky header on scroll
- Logo/name left, nav links right
- Active link: `color: var(--accent)`, no underline
- Hover: color shifts to `var(--accent)`, instant
- No hamburger menu required at MVP (links fit in one line)

### Code blocks

```css
background: var(--surface);
border: 1px solid var(--border);
border-radius: 6px;
padding: 16px 20px;
font: 400 13px/1.7 var(--font-code);
color: var(--text);
overflow-x: auto;
```

Shiki handles syntax highlighting. Configure with themes matching this palette. Languages: `javascript`, `typescript`, `sql`, `csharp`, `asm`.

### Form inputs

```css
background: var(--surface);
border: 1px solid var(--border);
border-radius: 4px;
color: var(--text);
font: 400 16px/1 var(--font-body);
padding: 10px 14px;

/* Focus */
border-color: var(--accent);
outline: 2px solid rgba(212, 134, 58, 0.3);
```

## What to Copy Into Astro

When scaffolding the Astro project, drop the CSS variables block into `src/styles/global.css` and import it in `src/layouts/BaseLayout.astro`. Import the Google Fonts `<link>` tags in the `<head>` of `BaseLayout.astro`.

Never hardcode hex values in component styles — always reference the CSS variables. This makes future palette tweaks a single-file change.
